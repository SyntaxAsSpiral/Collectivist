# Agent Context: Collectivist

AI-powered collection curator that transforms directories into living documentation substrates.

## System Architecture
- **Portable**: `.collection/` folder contains complete system
- **LLM-powered**: Uses local/cloud AI for analysis and description generation
- **Plugin-based**: Domain-specific scanners for different collection types
- **Multi-stage pipeline**: Analyzer → Scanner → Describer → Renderer

## Production Structure
```
user-collection/
├── .collection/              # Self-contained system
│   ├── src/                 # Python modules
│   ├── plugins/             # Collection scanners
│   ├── llm-config.yaml      # LLM configuration (pre-configured for LMStudio)
│   └── initialize-collection.* # Setup scripts
├── [user content files]     # Files to be organized
└── Collection.md            # Generated documentation
```

## Core Commands
```bash
python .collection/src/__main__.py analyze      # Detect collection type
python .collection/src/__main__.py scan         # Extract metadata
python .collection/src/__main__.py describe     # Generate AI descriptions
python .collection/src/__main__.py render       # Create documentation
python .collection/src/__main__.py update       # Full pipeline
```

## Configuration
- **llm-config.yaml**: LLM provider settings (defaults to free LMStudio)
- **collection.yaml**: Collection metadata (generated by analyzer)
- **collection-index.yaml**: Item index with descriptions (generated by scanner)

## LLM Providers
- **LMStudio**: Local, free, pre-configured default
- **OpenRouter**: Cloud, requires API key
- **OpenAI/Anthropic**: Cloud, requires API key
- **Ollama**: Local, free alternative

## Common Issues
- "Cannot reach LLM endpoint": LMStudio not running or config error
- "No collection.yaml found": Run analyze command first
- Import errors: Missing dependencies (requests, pyyaml)
- Path errors: Wrong working directory

## Expected Outputs
- **Collection.md**: Markdown documentation with AI descriptions
- **Collection.html**: Interactive HTML index
- **index.json**: Machine-readable export
